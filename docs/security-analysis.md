# EasyNet 安全性与防探测分析指南

本文档基于 EasyNet 项目当前的架构实现，对五种支持的代理协议在对抗防火长城（GFW）的深度包检测（DPI）、主动探测以及黑客端口扫描的能力进行了详细分析，并给出了推荐的安全等级排序。

---

## 推荐等级与排序 (按安全性从高到低)

### 🥇 级别一：终极隐蔽（极难被识别与封锁）

#### 1. Xray + Reality (VLESS-Reality-Vision)
*   **防探测原理**：目前代理界的天花板。Reality 协议完全摒弃了传统的服务端证书，它通过拦截客户端与**真实存在的大型网站**（如 `www.microsoft.com`）之间的 TLS 握手，在不持有目标网站私钥的情况下，将流量伪装成访问这些名站的正常流量。
*   **抗 GFW 能力**：**极高**。GFW 的 DPI 设备看到的完全是访问微软或苹果的合法 TLS 1.3 流量。如果 GFW 发起主动探测（重放攻击或扫描端口），Xray 会直接将探测流量引流给真实的微软服务器，GFW 收到的是微软真实的网页响应，完全无法区分这是代理。
*   **黑客防御**：黑客扫描该端口（默认 8443），看到的只是一个正常的微软反代服务，没有任何代理特征。

### 🥈 级别二：标准 HTTPS 伪装（主流抗封锁方案）

#### 2. Trojan-Go (配合 Nginx 随机路径回落)
*   **防探测原理**：核心设计理念是“将流量伪装成最常见的 HTTPS 流量”。在我们的项目中，它绑定在 443 端口，使用了你自己的真实域名和 TLS 证书。
*   **抗 GFW 能力**：**高**。当客户端发起带有正确密码和**随机 WebSocket 路径**（如 `/a6d31173`）的请求时，Trojan-Go 才处理代理。如果 GFW 或黑客发起主动探测（不带密码、路径错误、甚至非 TLS 流量），Trojan-Go 会立刻将这些流量交给后端的 Nginx 伪装站。GFW 看到的只是一个返回 404 的普通 Nginx 网页。
*   **项目安全加成**：我们特意将 Nginx 限制在 `127.0.0.1` 监听，防止了黑客绕过 Trojan 直接攻击 Nginx。同时随机生成的路径极大地增加了 GFW 穷举探测的难度。

#### 3. V2Ray (VMess + WebSocket + TLS)
*   **防探测原理**：和 Trojan-Go 类似，也是通过 WebSocket 将流量封装在 TLS 加密隧道中。在 EasyNet 架构中，它被巧妙地隐藏在 Trojan-Go 后面（端口多路复用）。
*   **抗 GFW 能力**：**较高**。由于外层套了 TLS，GFW 无法看到内部的 VMess 特征。结合随机路径，它能有效抵抗扫描。
*   **排名原因**：VMess 协议本身的握手过程比 Trojan 复杂且存在理论上的加密特征（尽管被 TLS 包裹）。但在当前“Trojan-Go 负责前置 TLS 解密和回落”的架构下，它的安全性依然非常可靠。

### 🥉 级别三：传统加密（存在被识别风险，适合特定场景）

#### 4. Shadowsocks (chacha20-ietf-poly1305)
*   **防探测原理**：基于 AEAD 的流式加密算法，没有握手过程（0-RTT），数据发过来直接就是加密后的乱码。
*   **抗 GFW 能力**：**中等/偏低**。虽然其使用的加密算法很安全，但其**“没有任何特征的完全随机数据流”本身就是一种特征**。GFW 的机器学习模型目前已经能够较高概率地识别出这种“全随机”的 SS 流量并进行阻断。
*   **黑客防御**：由于 SS 对非法请求没有任何回应（直接丢弃），在一定程度上防住了无脑的端口扫描，但针对性的重放攻击依然可能导致其暴露。

#### 5. WireGuard
*   **防探测原理**：基于 UDP 的无状态加密隧道。其密码学安全性极高，使用了现代的 Noise 协议框架。
*   **抗 GFW 能力**：**低（在特定跨国环境下）**。WireGuard 的设计初衷是“企业组网”而非“翻墙”。它的 UDP 数据包具有**非常明显且固定的协议头部特征**。GFW 几乎可以 100% 精准识别标准的跨国 WireGuard 握手包。
*   **实际表现**：在国内直接访问海外节点时，纯 WireGuard 流量极易遭到 GFW 的 UDP 阻断或限速。
*   **适用场景**：作为 VPS 之间的数据回源隧道，或者在某些尚未部署严格 UDP 审查的网络环境下使用。

## 💡 使用建议

在 EasyNet 项目生成的**节点订阅**中，节点列表正是按照上述安全等级从高到低排列的：

1.  **首选日常翻墙方案**：请优先使用位于订阅列表顶部的 **Xray+Reality** 或 **Trojan-Go** 节点，它们能提供最稳定、最抗封锁的体验。
2.  **备用方案**：如果首选节点遇到特定客户端兼容性问题，可以使用 **V2Ray** 节点。
3.  **特定场景使用**：底部的 **Shadowsocks** 和 **WireGuard** 节点建议仅在特殊网络环境（如国际中转加速）或测试时使用，不建议作为高危时期的主力翻墙节点。
