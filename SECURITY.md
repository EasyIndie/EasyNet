# 安全策略

## 受支持版本

仅最新稳定版接收安全补丁。不对旧版本回溯修复。

| 版本 | 支持状态 |
|------|----------|
| 最新版 | ✅ 是 |
| 旧版本 | ❌ 否 |

## 报告漏洞

请通过邮件直接联系项目维护者报告安全漏洞 — **请勿** 公开提交 GitHub Issue。

你应在 **48 小时** 内收到确认回复。如未收到，请跟进确认消息已送达。

### 建议格式

为帮助我们快速响应，请包含：

- 漏洞简要描述
- 复现步骤（最好附带概念验证）
- 受影响版本
- 潜在影响

### 处理流程

1. **确认** — 48 小时内
2. **调查** — 7 天内给出修复计划
3. **协同披露** — 修复发布后，在版本发布说明中披露漏洞

感谢你帮助保护 EasyNet 及其用户的安全。
